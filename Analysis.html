Jarvis March Time Complexity

The inner loop checks every point in the set S, and the outer loop repeats for each point on the hull. Hence the total run time is O(nh). The run time depends on the size of the output, so Jarvis's march is an output-sensitive algorithm.However, because the running time depends linearly on the number of hull vertices, it is only faster than O(nlog n) algorithms such as Graham scan when the number h of hull vertices is smaller than log n.

KirkPatrick-Seidel Time Complexity

The Algorithm UPPER-HULL correctly determines the sequence of vertices on the upper hull of S in O(n) space and O(n log H) time, where H is the number of edges on the upper hull of S.The function BRIDGE correctly determines the left and right bridge point in O( n) worst case time and space.The convex hull of a set of n points in the plane can be found in time O(n log H) using O(n) space, where H is the number of vertices found to be on the hull
